1. Открытие браузера. Пользователь открывает браузер на мобильном устройстве;

2. Нажатие на строку поиска. Пользователь нажимет на строку ввода текста для ввода запроса;

3. Ввод необходимого запроса. Пользователь вводит в строку поиска необходимый запрос, например доставка пиццы;

4. Возможно появление капчи:

	4.1 Капча появилась:

		4.1.1 Пользователь проходит капчу и продолжает поиск;

		4.1.2 Пользователь обновляет страницу и продолжает поиск.

	4.2 Капча не появилась.

5. Открытие страницы с результатами поиска. Открывается с страница с выдачей;

6. Запрос имеет ошибку:
	
	6.1 Если запрос был введен с ошибкой или на транслитом, то браузер предложит поиск с исправлением:

		6.1.1 Если ошибка меняет суть запроса то пользователь нажмет на предложенное браузером исправление;

		6.1.2 Если ошибка не меняет суть запроса то пользователь может продолжить поиск.

	6.2 Если запрос был введен правильно то пользователь начнет просмотр(скролинг) страницы с выдачей.

7. Скролинг выдачи:

	7.1 Просмотр рекламных предложений. В большенстве поисковиков первые предложенные выдачи являются рекламными и выставляются первыми при выдаче:

		7.1.1 Рекламные предложения удовлетворяют требованию пользователя. Пользователь выбирает выдачу из предложенных рекламных;

		7.1.2 Рекламные предложения не удовлетворяют требованию пользователя. Пользователь пропускает рекламные выдачи.

	7.2 Просмотр карты. Если активка геолокация то может присутвовать выдача с картой, где показаны ближайшие места доставки или тп;

		7.2.1 Открытие карты:

			7.2.1.1 Пользователь может выбрать необходимый ресурс (магазин, ресторан и тп) на карте, те скролинг по карте.

		7.2.2 Пропуск выдачи с картой.

	7.3 Просмотр остальных выдач:

		7.3.1 Предложенная выдача удовлетворяет требованию пользователя. Пользователь откроет эту выдачу;

		7.3.2 Предложение не удовлетворяет требованию пользователя. Пользователь продолжит скролинг.
		
	7.4 Конец страницы с выдачей:
	
		7.4.1 Дойдя до конца страницы с выдачей есть возможность перейте на след или загрузить еще. Пользователь перейдет на след страницу или загрузит еще вырианты выдачей.
		
		7.4.2 Дойдя до конца страницы с выдачей нет возможности перейти на след или загрузить еще. Если пользователь не нашел выдачу удовлетворяющую его требованиям, он либо закроет браузер либо поменяет суть запрора.

8. Открытие сайты:

	8.1 Сайт может быть агрегатором. Если сайт является агрегатором пользователь выбирает пункт (магазин, ресторан, кафе, тп) откуда придет заказ;

	8.2 Сайт не является агрегатором. Выдача переводит на сайт конкретного ресурса (магазина, ресторана и тп):

		8.2.1 Открывается страница с выбором товара. Если сразу открывается страница с выбором товара то пользователь может сразу приступить к его выбору.

		8.2.2 Открывается страница не с выбором товара. Если открывает главная страница ресурса, то пользователь должен найти и перейти на страницу с выбором товара.

9. Оформление заказа:

	9.1 Выбор необходимых критериев заказа (размер пиццы например);

	9.2 Указывание адресса доставки;

	9.3 Указывание способа оплаты;

	9.4 Указывание контактов для связи (имя, телефон, тп).

10. Оплата:

	10.1 Оплата онлайн. Если был выбран способ оплаты онлайн (карта, эл. кошелек, криптовалюта), то пользователь должен сначала оплатить заказ;

	10.2 Оплата оффлайн. Если был выбран способ оплаты оффлайн (наличными, картой при получении), то пользоватлеь сразу переходит к подтверждению заказа.

11. Подверждение заказа. Пользователь на нажимает на кнопку подтверждения заказа и переходит на страницу с его данными (номер заказа и тп).

12. Закрытие браузера. Пользователь закрывает браузер удобным ему способом.



Исключения:

1. Отсутствие подключения к интернету. Перед каждым открытием новой страницы необходимо проверять наличие подключнеия к сети.

	1.1 Подключение отсутствует. Если при попытке открытия страницы она не открылась, то пользователь будет обновлять страницу с определенным интервалом (5с например) или определенное количество раз (n), пока страница не откроется.

	1.2 Подключение присутствует. Страница откроется без проблем.
